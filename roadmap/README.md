# Roadmap

The roadmap is split one-feature-per-file across three directories:

- `planned/` — not started
- `in-progress/` — started, partially done
- `completed/` — shipped

Status is encoded by directory, so there are no status emoji to go stale. To add a new feature, create a file in the right directory. To update status, move the file between directories.

`backlog.md` holds loose tasks that don't belong to any single in-progress feature.
